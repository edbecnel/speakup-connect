Translation Helper

Edit and approve UI strings for your organization’s languages

Sign in

Platform super_admin, org admin, or manageTranslations permission.

After sign-in, open Translation workspace help or click Help for editing instructions.

Translation workspace help

English is the source language. Your organization maintains other UI languages here (for example Bisaya / Cebuano or Tagalog). End users only pick which language they see in the mobile app — they do not edit strings here.

Quick start (translation moderator)

  1. Select the Target locale for the language you are editing.
  2. Use Filters (Key, English, Target, Status), or Feature to find strings.
  3. Read the English column, then type your translation in the Target field.
  4. Click Save to store as in review, or Approve when the wording is final.
  5. Optional: AI draft on one row for a suggested first pass — always review before approving.

Assign translation moderators

Org admins only. Delegates can edit translations without full admin access. In the mobile app: Settings → Admin Dashboard → Roles & Permissions. Create or edit a role, enable Translation moderator (edit UI strings) (manageTranslations), save, then Assign the role to an approved member. They will see Settings → Administration → Translations in the app.

Human translator workflow (Google Sheets)

  1. Select the target locale and click Export CSV.
  2. Upload the CSV to Google Sheets (or Excel). Share with your translator.
  3. Translator edits the translation column only; keep key and ICU placeholders unchanged.
  4. Download the sheet as CSV (File → Download → Comma-separated values in Google Sheets).
  5. Click Import CSV, choose the file, confirm, then review rows in the workspace.
  6. Org admin: Export ARB when translations are approved for a release.

Toolbar reference

Target locale
Languages enabled for your organization. You cannot edit English here.
Feature
Filter by key prefix (for example settings, auth).
Filters
Filter by Key, English, Target, or Status (combine any of them). Use Clear filters to reset.
Refresh
Reload the list from the server.
Import app_en.arb
Platform super-admin only. Upload English source keys from lib/l10n/app_en.arb.
Approve all saved/in-review
Sets every in_review row (saved target, not yet approved) to approved in one step.
Save all AI drafts
Copies every ai_draft into the saved target as in_review so you can review and approve in bulk.
Translate missing (AI)
Org admin only. Batch AI drafts for all missing strings in the selected locale.
Export CSV
Download all strings as a spreadsheet-friendly CSV (key, english, translation, status). Open in Google Sheets for offline human review.
Import CSV
Upload an edited CSV to update target translations in the workspace. Rows need key and translation columns. Optional status: in_review (default) or approved. Unknown keys are skipped; placeholders like {name} must match English.
Export ARB
Org admin only. Download approved strings as ARB JSON for app releases.

Row actions

Save
Stores your edit with status in_review.
Approve
Marks the string approved and ready for export.
AI draft
Generates a machine suggestion for this row only. Review and edit before approving.

Status values

  • missing — no target text yet
  • ai_draft — AI suggestion present; needs human review
  • ai_draft_failed — AI could not draft (often placeholder mismatch); edit manually
  • in_review — human edit saved, not yet approved
  • approved — final; included in ARB export

Placeholder rules

Keep ICU placeholders exactly as they appear in English, for example {name}, {count}, or {error}. Do not translate or remove curly-brace placeholders or the app may show broken text.

Platform setup (deployment lead only)

Not for school org admins. One-time setup per Firebase environment before this tool and the in-app Translations workspace work.

  1. Deploy Cloud Functions: npx firebase-tools deploy --only functions:getTranslationWorkspaceAccess,functions:importTranslationSource,functions:importTranslationTargets,functions:listTranslationEntries,functions:saveTranslationEntry,functions:draftTranslation,functions:batchDraftTranslations,functions:exportTranslationArb
  2. Seed roles: node scripts/seed_roles.js (or add manageTranslations in Firestore manually).
  3. Optional AI: set secret TRANSLATION_AI_API_KEY.
  4. Platform super_admin: import lib/l10n/app_en.arb here; set ORGANIZATION_ID in firebase-config.js for org-scoped sign-in.

See tools/translation-helper/README.md and Help Center → Administrator Guide → Platform setup in the app.

The in-app editor is at Settings → Administration → Translations. Full administrator documentation is in the app Help Center → Administrator Guide.
