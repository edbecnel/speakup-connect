# Mobile-First Tutorial Quality Checklist

Purpose: ensure tutorials can be completed on mobile without requiring constant guide switching.

## Core principle

Tutorials must be self-sufficient for first-time users in-app. Guides are optional deep-dive references.

## Pass criteria

### 1) Self-sufficient flow
- [ ] Module can be completed without opening a guide.
- [ ] Required taps/paths are written in the tutorial.
- [ ] Guide links are optional "Learn more".

### 2) Clear audience and permissions
- [ ] Audience is explicit (member/admin/leader/staff).
- [ ] Capability requirements are inline when needed.
- [ ] Missing menu cases are explained.

### 3) Actionable steps
- [ ] Steps are sequential and concrete.
- [ ] UI labels match what users actually see.
- [ ] Ambiguous wording is avoided.

### 4) In-module troubleshooting
- [ ] Common blockers appear near the relevant step.
- [ ] Each blocker includes a quick fix path.

### 5) Mobile readability
- [ ] Sections are short and scannable.
- [ ] "Try this now" tasks are concise and testable.

### 6) Localization readiness
- [ ] Language-switching instructions are accurate.
- [ ] Fallback behavior is acceptable when localized tutorial is missing.

### 7) In-app validation
- [ ] Tutorial was tested in Help Center.
- [ ] No missing asset or broken-link issues.
